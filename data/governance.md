# Governance

## Risk Register

| Риск | Вероятность/Влияние | Детект | Защита | Остаточный риск |
|------|---------------------|--------|--------|------------------|
| Несанкционированное использование API (нарушение правил) | Средняя / Высокое | Мониторинг частоты запросов | Использовать официальные API с ключами, кешировать результаты | Низкий |
| Утечка резюме (персональные данные) | Низкая / Очень высокое | Аудит | Не хранить резюме после сессии, передавать по HTTPS | Средний |
| Генерация писем, содержащих ложную информацию | Средняя / Среднее | Проверка фактов по резюме | Запретить генерацию утверждений об опыте, которого нет в резюме | Низкий |
| Предвзятость при сравнении кандидатов | Низкая / Среднее | Аудит логов сравнения | Использовать объективные критерии (навыки, годы), не учитывать имя/пол | Низкий |

## Политика логов и работы с данными
- Резюме обрабатывается в памяти, не сохраняется на диск.
- Логи не содержат контактной информации (телефон, email заменяются хешем).
- Внешние API вызываются с минимально необходимыми параметрами.

## Защита от инъекций и подтверждение действий
- Поля ввода при поиске вакансий экранируются (чтобы избежать SQL-подобных инъекций в запросы API).
- Перед отправкой сгенерированного письма пользователь должен явно подтвердить, что проверил его.
