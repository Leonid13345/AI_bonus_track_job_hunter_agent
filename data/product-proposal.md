# Product Proposal: Job Hunter Agent

## Обоснование идеи
Прикладная задача – сократить время на поиск работы и повысить качество откликов за счёт анализа соответствия и персонализации. Агент выступает карьерным консультантом.

## Цель и метрики
- **Цель**: прототип, который за 2–3 диалога помогает пользователю найти 5 подходящих вакансий и подготовить 1 качественное сопроводительное письмо.
- **Метрики успеха**:
  - Продуктовые: доля вакансий, которые пользователь счёл релевантными (>70%), конверсия в отклики (если отследим).
  - Агентские: точность извлечения требований из вакансии (сравнение с ручным анализом), качество сгенерированных писем (оценка пользователем).
  - Технические: время обработки вакансии <3 сек, доступность API источника.

## Сценарии использования (включая edge‑кейсы)
- Основной: пользователь загружает резюме и просит "найди вакансии Java разработчика в Москве с зарплатой от 200к".
- Анализ: "почему меня не берут?" → агент ищет слабые места.
- Edge‑кейсы:
  - Вакансия с неполным описанием → агент запрашивает уточнение или пропускает.
  - Противоречивые требования (например, "опыт 5 лет и выпускник") → отмечает как потенциальный рефлаг.
  - Резюме не содержит ключевых навыков → агент предлагает добавить.

## Ограничения
- **Технические**: p95 latency <10 сек (при парсинге нескольких вакансий), квота API.
- **Операционные**: соблюдение robots.txt и условий использования API.

## Архитектурный набросок
- **Модули**:
  - Resume Parser (извлечение структуры из PDF/текста)
  - Job Fetcher (HH.ru API, LinkedIn API – если доступен)
  - Matcher (сравнение навыков, опыта)
  - Letter Generator
  - Interview Coach
- **Интеграции**: API досок вакансий, возможно, OpenAI для генерации текстов.
- **LLM**: для извлечения требований, генерации писем, симуляции интервью.

## Data Flow
1. Загрузка резюме → Resume Parser → структурированный профиль.
2. Пользователь задаёт критерии → Job Fetcher получает вакансии.
3. Для каждой вакансии Matcher вычисляет score.
4. По запросу Letter Generator создаёт сопроводительное письмо.
5. Пользователь может уточнять, сохранять в список отслеживания.

**Что делегируется LLM**: анализ текста, генерация, извлечение сущностей.
**Что НЕ делегируется**: хранение резюме, отправка писем, доступ к аккаунту пользователя.
